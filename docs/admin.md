# Admin管理后台完整使用手册  

使用管理后台之前，请先配置并安装好PhalApi专业版，并创建超级管理员账号。  

> 适合阅读对象：管理员。  

# 登录   

## 访问管理后台

假设配置的域名是 [http://open.phalapi.net](http://open.phalapi.net)，那么管理后台的地址是： 

```
http://open.phalapi.net/admin/
```

或者通过网站的顶部菜单，点击【管理后台】直接进。   

> 温馨提示：出于安全考虑，建议在完成私有部署后，手动隐藏或注释管理后台的访问菜单。  

## 登录管理后台

打开管理后台，首先需要进行管理员登录，登录界面如下：  

![](./img/yesyesapi_20200426164442_898e22552eb1fbbbb82cf118b20eac3e.png)  

管理员账号密码，在安装向导时，由用户自行设定。如果忘记管理员账号，可查看 ```pp_member``` 数据库表；如果忘记密码，需要重新安装。  


# 后台首页  

进入管理后台后，可以看到类似以下的后台首页。其他功能模块，按界面指引操作即可。  

在后台首页，可以快速查看诸如：待审核应用、全部账号、今日接口请求次数、全部接口数量等整体概况。以及：接口流量统计图表、近期统计表格数据、昨日活跃App和系统授权信息等。  

![](./img/20221205-224644.png)  

其中，首页几个统计数据的口径是：    

 + 接口流量统计：根据接口请求日记实时统计最近30天的接口流量（排除开放平台和管理后台接口请求）  
 + 近期统计：按每日统计，包括：日期、订单总数量、应用总数量、用户总数量、活跃应用数量、UV、请求次数  
 + 昨日活跃App：包括：日期、App Key、活跃应用数量、UV、请求次数  

术语解释：  

 + 用户总数量：开发者账号数量+应用会员数量+管理员数量  
 + 活跃应用数量：当天有接口请求的App Key去重后数量    
 + UV：通过OpenAPI调用时所绑定关联的登录会员去重后数量  
 + 请求次数：指OpenAPI接口请求次数（不分成功与否，排除开放平台和管理后台接口请求）  

> 温馨提示：除了接口请求日记外，如无特殊说明，其他全部统计报表，都以 **OpenAPI接口请求** 为统计口径和范围，即 **排除开放平台和管理后台接口请求**。  

点击左侧功能菜单可以展开折叠菜单，和点击进入具体的功能界面。  

下面按功能模块简单介绍管理后台的功能和注意事项，最新的界面以最新版的为准。 

# API接口计费  

此模块主要功能有：  

 + **接口流量套餐：**  
 支持试套餐和付费套餐，接口流量套餐列表、添加套餐、修改套餐、删除套餐，支持组合套餐；  

 + **订单管理：**  
 订单列表、修改订单、套餐调整；  

 + **套餐管理：**  
 套餐列表、套餐发放、调整套餐、删除套餐；  

 + **支付配置：**  
 支付宝配置、PayPal配置、现金支付配置。  

## 接口流量套餐管理

在接口流量套餐，可以查看已经配置的接口服务套餐，主要是针对：免费试用、付费购买、组合套餐，以及对套餐的检索、编辑和删除等操作。  

![](./img/20221205-225402.png)  

> 特别注意：默认情况下，全部OpenAPI接口在分配接口权限后即可调用。若API接口有设置和开启 免费试用 或 付费购买 的套餐后，此接口则**必须**需要有 免费试用 或 付费购买流量 或 后台发放的流量，才可进行调用。  

> 温馨提示：删除已有的套餐，不会影响客户已经成功购买的套餐使用。  

### 添加新套餐  

当需要添加新的接口套餐，可以点击【添加套餐】菜单入口或页面按钮，进入到 添加套餐 的界面。  

![](./img/20221205-225715.png)  

在设置接口流量套餐时，你可以设定以下套餐信息：  

 + 类型（付费购买/免费试用/组合套餐）  
 + 套餐名称（中文展示名称）  
 + 接口服务（需要选择已经成功上架和发布的OpenAPI接口）  
 + 接口流量（单位：次，本次的接口流量的可用次数）  
 + 套餐价格（单位：元，保留两位小数点）  
 + 有效时间（可选：1个月/2个月/3个月/6个月/1年/2年，支持可配置）  
 + 套餐说明（支持富文本编辑，下单时展示给客户）  
 + 状态（正常/关闭，关闭后，前台将不会展示购买入口）  

### 套餐类型说明  

接口流量套餐，主要分为三种类型：免费试用、付费购买、组合套餐。其区别是：  

 + **免费试用：**  
 指客户不需要付费即可免费调用的次数，每份免费试用套餐按开发者账号进行自动分配，即每个开发者账号针对每一个API接口最多享受一次免费试用套餐（不区分应用）。同一个API接口，最多可设置一个免费试用套餐，也可以不设置。  

 + **付费购买：**  
 是指客户需要购买后才可以使用的接口流量套餐，价格不能为0.00元。客户可以重复购买，或提前购买。如果客户针对同一个API接口同时存在多份相同的接口流量套餐，系统将优先消耗即将过期的流量套餐。此外，付费购买的流量将比免费试用的流量优先消耗。  

 + **组合套餐：**  
 组合套餐，是出于运营和优惠活动的考虑而设定和提供的。你可以基于前面设置的多个付费购买套餐，进行再组合，以更优惠的打包价格提供给你的客户。

### 人工修改订单  

何时需要进行人工修改订单？  

以下几个场景，需要管理员在后台人工修改订单。  

 + 场景1：平台设定或允许现金支付和对公转账，则需要人工修改订单状态为：已支付，和进行接口流量套餐的发放。  

 + 场景2：内部手动调整订单价格，价格调整后，前台支付时则以管理后台的订单价格为准。  

 + 场景3：手动调整接口流量的次数，例如：额外赠送更多的接口次数。 

 + 场景4：其他有关订单的修改，例如：订单备注。

## 管理已购买的服务

针对客户已经购买的接口服务，或者对于已经发放的流量服务，进行搜索、查看、编辑、删除和管理。  

![](./img/20221205-232027.png)  

### 调整套餐  

点击 【调整套餐】，你可以修改两个重要的信息：接口总次数和有效期。  

 + **接口总次数：**  
 要注意到调整的套餐，可能已被客户使用。若调整的新总次数大于客户已使用次数，则流量包为**使用中**；若调整后的新总次数小于等于客户已使用次数，则流量包为**已用完**。  

 + **有效期：**  
 若调整的新有效期大于当前时间，则流量包有效；若小于当前时间，则流量包过期。  

### 套餐状态说明  

已购买的服务，其套餐状态分为：未使用、使用中、已用完、已过期、已失效。  

分别表示：  

 + 未使用（可以使用）：新购买或新发放，客户从未使用；  
 + 使用中（可以使用）：客户已经使用过一次或以上，但总次数未达到；  
 + 已用完：使用次数已经大于或等于总次数；   
 + 已过期：不管总次数是否用完，有效期已到；  
 + 已失效：订单取消或回收。

# API接口管理  

API接口管理，是平台的核心部分。可以进行接口权限分配和管理（权限分配）、进行可视化接口设计（接口开发）、接口自动化黑盒测试（接口测试）等。    

分为四大子模块：  

 + 低代码接口开发  
 + 接口权限  
 + 接口测试  
 + 接口监控  

下面依次展开介绍。  

## 低代码接口开发  

支持：自动生成API接口源代码，接口管理、在线开发、发布、自动生成，支持网关接口、数据库接口、普通接口生成模式、支持GET/POST方法设置，生成数据API接口时可以选择数据库和数据库表，同时支持 简易代码模式和完整代码模式。  

你也可以导入接口：支持swagger批量导入API接口，支持多个接口的勾选、覆盖导入和快速发布。  

进入 低代码接口开发列表，可以对已经发布或还处于草稿状态的API接口进行管理和查看。

![](./img/20221207-114136.png)  

### 添加新接口  

除了参考开发文档并且通过PHP源代码的方式编写API接口外，对于非技术编程人员，可以通过管理后台，创建新接口，完成界面化的接口低代码开发。  

接口低代码开发的步骤主要是：  

 + 第1步：接口设计  
 + 第2步：生成代码  
 + 第3步：发布接口  

![](./img/20221207-114436.png)  

### 接口设计

在接口编辑器界面，接口设计的信息表单，主要有：  

 + 接口设计类型（HTTP网关接口API/数据库表接口API/普通接口API）  
 + 接口服务名称（由 ```命名空间 + 接口类名 + 接口方法名``` 组成）  
 + 接口模块（对应接口文档左侧分类菜单）  
 + 接口服务标题  
 + 接口功能描述  
 + 隐藏接口文档（勾选后，前台不展示此接口文档，但依然可以正常调用API）  
 + 请求方式（支持：GET/POST）  
 + 接口参数（支持多个参数配置）  
 + 返回结果（支持多个结果配置）  
 + 简易代码模式/完整代码模式（主要区别在于：简易代码模式只维护接口核心函数，相对简单）  
 + 接口文件路径（PHP接口源文件的保存路径）  
 + 接口返回示例（用于展示）  

> 温馨提示：关于接口设计类型的说明和区别，HTTP网关接口API用于生成请求和封装上游API接口的调用，数据库表接口API可用于生成对应数据库、对应数据库表和常用CURD的基本接口生成。

## 接口权限  

此子模块提供了：  

 + **接口权限分配：**  
 接口权限分配，自动检测API接口资产，支持快速按配置的开发者角色进行接口权限分配，列表查看
权限预览：查看、预览和搜索指定应用的接口权限。  

 + **接口权限规则设置：**  
 支持接口权限的规则配置、添加、列表查看、搜索和删除、撤销等接口，权限规则支持按开发者角色、开发者账号、开发者应用共三个维度。

 + **接口服务操作列表：**  
 支持手动添加接口服务，一般情况下不需要用，为方便未来扩展自己的接口路径，特此预留。  

在开放API接口前，需要在管理后台，提前针对开发者角色、开发者账号、开发者应用这几个维度进行接口的权限分配，否则开发者将无权访问和调用指定的API接口。  

> 值得注意：为了方便调试和内部维护，管理员拥有最高接口权限，即使不分配任何接口权限，管理者账号名下的应用都可以调用任何API接口。 

## 接口限流

### 环境要求
> Redis服务和php-redis扩展

### 代码配置

./config/sys.php文件中新增redis配置
```php
    'redis' => array(
        //Redis链接配置项
        'servers'  => array(
            'host'   => '127.0.0.1',        //Redis服务器地址
            'port'   => '6379',             //Redis端口号
            'prefix' => 'pro_',                 //Redis-key前缀
            'auth'   => 'xxxxxx',                 //Redis链接密码
        ),
        //Redis分库对应关系操作时直接使用名称无需使用数字来切换Redis库
        'DB'       => array(
        ),
        //使用阻塞式读取队列时的等待时间单位/秒
        'blocking' => 5,
    ),
```

./config/di.php文件中引入PhalApi Redis服务
```php
// redis服务
$di->redis = new \PhalApi\Cache\RedisCache($di->config->get("sys.redis.servers"));
```
./config/app.php文件修改，增加限流总开关
```php
'limit_flow'=>[
        'switch' => true
    ]

```
**到此，限流服务已经准备完成，接下来便是对限流进行可视化配置**
### 限流可视化配置
```
限流配置规则（优先级从高到低）：
    开发者应用  > 开发者账号 > 开发者角色 > 开发者全局应用

举例：对开发者账号A进行限流100次/秒，对开发者A的某个应用AK1进行20次/秒限流，优先匹配AK1（20次/秒）的限流规则
    
```

1. 【管理后台】- 【Api接口管理】- 【接口规则权限】- 【创建新规则】

接口权限分配，可以基于现有的API接口给开发者角色（最大的维度）快速分配接口权限。

![](./img/20221207-122508.png)

> 温馨提示：不同维度的权限规则优先级是：开发者应用接口权限（最高） > 开发者账号接口权限 > 开发者角色接口权限（最低） > 全局默认的接口权限配置（PHP）。    

## 接口测试  

此子模块提供了：  

 + **接口测试：**  
 自动化接口测试，接口测试、全局access_token配置、搜索用例、添加新用例、运行用例、编辑查看用例、测试报告

 + 通过率：计算方式 = 成功用例 / 失败用例；  
 + 全部接口测试用例：当前用例总数量。

关于接口类型的说明：

点击【权限】-【管理后台权限分配】，可针对管理后台的功能操作，向管理员进行权限分配。  

![](./img/yesyesapi_20200514214829_fad6a5343e6e4346280d29d59a84a01a.png)  

> 温馨提示：只有超级管理员，才有权限进行权限分配。  

添加新操作，如果有新增的菜单，可添加新操作，然后再进行权限分配。  

![](./img/yesyesapi_20200514215130_bbbdcf4e1fca494cba0adf990a93fd6c.png)  

其中，URI链接为管理后台的页面路径，不需要域名，不需要```/admin/```，对应页面后面的路径。  

针对管理员角色分配管理后台菜单操作权限：  

![](./img/yesyesapi_20200514215601_0e4e053666c1882e6a2e9fd60102cb38.png)  

> 温馨提示：超级管理员拥有全部管理后台的权限，无须分配。  

也可以针对个别的账号ID分配管理后台菜单权限：  

![](./img/yesyesapi_20200514215725_d4c7e58d053971f2ab20129e2c2b7a62.png)

# 报表统计 

报表统计，是针对开放平台，提供了核心的业务指标统计。此模块主要提供了：  

 + **应用历史统计：**  
 统计每个应用每日的汇总数据，支持查看付费接口、试用接口、普通接口、扣费失败接口请求统计；  

 + **每日注册统计：**  
 每日在开放平台账号注册人数统计及同比上个月，拆线图，支持日期范围搜索，统计表格；  

 + **每日登录统计：**  
 每日在开放平台及管理后台账号登录成功与失败的统计及同比上个月，拆线图，支持日期范围搜索，统计表格。  

## 应用历史统计

统计每个应用每日的汇总数据，支持查看付费接口、试用接口、普通接口、扣费失败接口请求统计。支持时间范围的统计，依赖于每日定时统计的报表数据。  

相关统计口径，前文有介绍，不再赘述。  

![](./img/20221207-224051.png)  


## 每日注册统计

每日在开放平台账号注册人数统计及同比上个月，拆线图，支持日期范围搜索，统计表格。  

![](./img/20221207-224256.png)  

默认查询最近7天，可以同比上个月。  


## 每日登录统计

每日在开放平台及管理后台账号登录成功与失败的统计及同比上个月，拆线图，支持日期范围搜索，统计表格。   

![](./img/20221207-224329.png)  

统计字段口径说明：  

 + 总登录次数：计算公式 = 成功登录次数 + 失败登录次数  
 + 成功登录次数：成功登录开放平台、管理后台和开放平台，以及通过OpenAPI登录的成功人次；    
 + 失败登录次数：登录开放平台、管理后台和开放平台，以及通过OpenAPI登录的失败人次。    

> 温馨提示：如果在二次开发过程中，使用了```\Base\Domain\Login::addLoginLog('登录账号', \Base\Domain\Login::LOGIN_STATUS_SUCCESS);```，也会纳入图表统计。  

# 我的

作为管理员个人的管理功能，有：  

 + 个人资料（查看和个人资料修改）  
 + 最近登录（查看本账号最近登录时间和登录IP）  
 + 修改密码  
 + 退出登录  
 + 查看内部接口文档（为方便开发，管理员可以查看开放平台和管理后台和计划任务的API在线接口文档）
